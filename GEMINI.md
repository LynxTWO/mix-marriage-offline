# GEMINI.md - MMO Compatibility Notes

`AGENTS.md` is the authoritative steering file for AI-assisted work in this
repo.

Gemini sessions must follow `AGENTS.md` for:

- product principles and standing promises
- working modes and approval gates
- coverage-ledger rules
- unknown and evidence handling
- writing rules for comments, commits, docs, and PRs
- sensitive-data handling
- PR close-out expectations

If this file and `AGENTS.md` disagree, follow `AGENTS.md`.

## Read first

1. `PROJECT_WHEN_COMPLETE.md`
2. `AGENTS.md`

## Gemini-specific startup notes

- Treat `AGENTS.md` as the workflow and policy contract.
- Source repo-specific implementation heuristics from the docs, schemas,
  ontology, and backend contracts instead of restating them here.
- Use `docs/architecture/system-map.md` for runtime units and trust boundaries.
- Use `docs/semantic_contracts.md` when a field or artifact looks repetitive
  and you need to know what is canonical.
- Use `PROJECT_WHEN_COMPLETE.md` for product promises, completion posture, and
  release-truth constraints.

This file is a startup companion, not a second steering surface.
